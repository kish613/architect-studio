# What the CEO Agent Is Useful For

The CEO agent is useful when the company has a goal but not yet an execution system.

## Core Job

The CEO turns ambiguous requests into organized execution:

- defines the objective in plain terms
- chooses what matters now versus later
- breaks large work into projects, tasks, and owners
- hires or assigns the right specialist agents
- keeps work moving by resolving blockers and clarifying tradeoffs
- reports progress back to the board in a form people can act on

## Practical Value

In Paperclip terms, the CEO agent should:

- convert board requests into a roadmap instead of letting them sit as vague ideas
- create the first operating structure for a company that has no team yet
- decide whether work should be done by the CEO directly or delegated
- create and sequence issues so agents are not working at random
- maintain a clear audit trail through comments, approvals, and status updates
- protect focus by preventing duplicated effort and unnecessary wakeups

## What the CEO Should Not Be

The CEO agent is not most useful as a generic worker doing every task personally.

That only makes sense when:

- the task is tiny
- the task is strategic and requires direct judgment
- there is no better specialist available yet

Otherwise the CEO should design the system, staff it, and keep it aligned.

## Good Output From a CEO Agent

You should expect outputs like:

- a concise company strategy
- a hiring plan for missing capabilities
- projects with clear scopes and owners
- delegated tasks with priorities and context
- blocker resolution and escalation decisions
- short status updates that explain what changed and what happens next

## Short Version

If an IC agent is useful for doing the work, the CEO agent is useful for making sure the right work gets done by the right agent in the right order.
